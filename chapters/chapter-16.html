Deployment to Production

In This Chapter

  • Preparing for production
  • Environment variables with python-decouple
  • Configuring production settings
  • Creating build scripts
  • Deploying to Render

Your app is ready — let's deploy it to the web! We'll configure production settings and deploy to Render.

Installing Production Dependencies

Install the production dependencies:

Terminal
pip install gunicorn
Terminal
pip install whitenoise
Terminal
pip install dj-database-url
Terminal
pip install python-decouple
PackagePurpose
gunicornProduction WSGI server (replaces runserver)
whitenoiseServes static files efficiently in production
dj-database-urlParses database URLs (for cloud databases)
python-decoupleReads environment variables from .env files

Environment Variables

Create a .env file in your project root (never commit this to git!):

.env
# .env
SECRET_KEY=your-super-secret-key-here
DEBUG=False
ALLOWED_HOSTS=.onrender.com,localhost,127.0.0.1
DATABASE_URL=sqlite:///db.sqlite3

Production Settings

Update your settings.py for production:

settings.py
# settings.py
from decouple import config
import dj_database_url

SECRET_KEY = config('SECRET_KEY')
DEBUG = config('DEBUG', default=False, cast=bool)
ALLOWED_HOSTS = config('ALLOWED_HOSTS', default='').split(',')

# Database
DATABASES = {{
    'default': dj_database_url.config(
        default=config('DATABASE_URL', default='sqlite:///db.sqlite3')
    )
}}

# Static files with WhiteNoise
MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'whitenoise.middleware.WhiteNoiseMiddleware',  # Add after SecurityMiddleware
    # ... rest of middleware ...
]

STATIC_URL = '/static/'
STATIC_ROOT = BASE_DIR / 'staticfiles'
STATICFILES_STORAGE = 'whitenoise.storage.CompressedManifestStaticFilesStorage'

Create requirements.txt

Terminal
pip freeze > requirements.txt

Build Script

Create a build.sh script for Render:

build.bat
# build.bat — Run during deployment

pip install -r requirements.txt

python manage.py collectstatic --no-input
python manage.py migrate

Create .gitignore

.gitignore
# .gitignore
__pycache__/
*.py[cod]
*.sqlite3
.env
venv/
media/
staticfiles/
*.log

Deploying to Render

Follow these steps to deploy your app:

  1. Push to GitHub

    Run these commands to initialize and push your project to GitHub:

    Terminal
    git init
    Terminal
    git add .
    Terminal
    git commit -m 'Initial commit'
    Terminal
    git remote add origin https://github.com/yourusername/yourproject.git
    Terminal
    git push -u origin main
  2. Create a Render account — Sign up at render.com and connect your GitHub account.
  3. Create a new Web Service
    • Select your repository
    • Build command: build.bat
    • Start command: gunicorn helloworld.wsgi:application
  4. Set environment variables in the Render dashboard:
    • SECRET_KEY — Generate a new one
    • DEBUG — False
    • ALLOWED_HOSTS — .onrender.com
    • DATABASE_URL — Your database URL (if using PostgreSQL)
  5. Deploy! — Render will automatically build and deploy your app.
