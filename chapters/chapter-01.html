Getting Started — Setting Up Django

In This Chapter

  • Opening a terminal
  • Creating a virtual environment
  • Installing Django
  • Starting your first project
  • Understanding the project structure
  • Running the development server

Let's set up your development environment and create your very first Django project.

Opening a Terminal

Most of the work in Django development happens in the terminal (also called command line or command prompt). Here's how to open it:

  • Windows: Press Win + R, type cmd or powershell, and press Enter. Alternatively, search for "Command Prompt" or "PowerShell" in the Start menu.
  • macOS: Press Cmd + Space to open Spotlight, type "Terminal", and press Enter.
  • Linux: Press Ctrl + Alt + T or search for "Terminal" in your applications menu.

The terminal is where you'll run commands to create projects, install packages, and start your development server.

Creating a Virtual Environment

Before installing Django, it's best practice to create a virtual environment. This keeps your project dependencies isolated from your system Python installation.

Open your terminal and run:

Terminal
mkdir django-projects\ncd django-projects\npython -m venv venv

Now activate the virtual environment. Use the command that matches your shell:

Terminal
# On macOS/Linux:
source venv/bin/activate

# On Windows Command Prompt:
venv\Scripts\activate

# On Windows PowerShell:
.\venv\Scripts\Activate.ps1

You'll see (venv) appear at the beginning of your terminal prompt, indicating the virtual environment is active.

Why virtual environments? Without one, installing packages affects your entire system. Virtual environments let each project have its own dependencies at specific versions.

Installing Django

With your virtual environment active, install Django using pip:

Terminal
pip install django

Verify the installation:

Terminal
python -m django --version

You should see a version number like 5.1 or higher.

Starting Your First Project

Django provides a command-line tool called django-admin that helps you scaffold new projects. Let's create one called helloworld:

Terminal
django-admin startproject helloworld

This creates a new directory with the following structure:

Project Structure
helloworld/
├── helloworld/
│   ├── __init__.py
│   ├── asgi.py
│   ├── settings.py
│   ├── urls.py
│   └── wsgi.py
└── manage.py

Understanding the Project Structure

Let's break down what each file does:

FilePurpose
manage.pyCommand-line utility for interacting with your project (running server, migrations, etc.)
settings.pyConfiguration for your Django project (database, installed apps, middleware)
urls.pyThe URL routing configuration — maps URLs to views
wsgi.pyEntry point for WSGI-compatible web servers (production)
asgi.pyEntry point for ASGI-compatible web servers (async support)
__init__.pyTells Python this directory is a package

Running the Development Server

Navigate into your project directory and start the development server:

Terminal
cd helloworld\npython manage.py runserver

You'll see output like this:

Terminal Output
Watching for file changes with StatReloader\nPerforming system checks...\n\nSystem check identified no issues (0 silenced).\n...\nStarting development server at http://127.0.0.1:8000/\nQuit the server with CONTROL-C.

Open your browser and go to http://127.0.0.1:8000/. You should see Django's welcome page with a rocket ship — congratulations, your first Django project is running!

Tip: The development server auto-reloads when you save changes. You don't need to restart it every time you edit code.
