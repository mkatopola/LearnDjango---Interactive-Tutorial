Django Admin Panel

In This Chapter

  • Registering models with the admin
  • Creating a superuser
  • Customizing the admin display
  • Admin search and filters

Django's built-in admin panel is one of its killer features. It gives you a fully functional content management interface with just a few lines of code.

Registering Models with the Admin

To make a model appear in the admin panel, register it in your app's admin.py:

blog/admin.py
# blog/admin.py
from django.contrib import admin
from .models import Post

admin.site.register(Post)

That's it — one line of code and you get a full CRUD interface for your model.

Creating a Superuser

You need an admin account to access the panel. Create one with:

Terminal
python manage.py createsuperuser

Follow the prompts to set a username, email, and password. Then visit http://127.0.0.1:8000/admin/ and log in.

What the Admin Panel Provides

Out of the box, the admin gives you:

  • List view — See all objects in a table
  • Add/Edit forms — Create and modify objects through auto-generated forms
  • Delete — Remove objects with confirmation
  • Search — Find objects quickly
  • Filtering — Filter by field values
  • Permissions — Control who can do what

Customizing the Admin Display

For more control, use a ModelAdmin class:

blog/admin.py
# blog/admin.py
from django.contrib import admin
from .models import Post

@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    list_display = ('title', 'author', 'is_published', 'created_at')
    list_filter = ('is_published', 'created_at')
    search_fields = ('title', 'body')
    list_editable = ('is_published',)
    ordering = ('-created_at',)
    date_hierarchy = 'created_at'
    prepopulated_fields = {{'slug': ('title',)}}
OptionWhat It Does
list_displayColumns shown in the list view
list_filterSidebar filters
search_fieldsFields included in the search bar
list_editableFields editable directly in the list view
orderingDefault sort order
date_hierarchyDate-based drill-down navigation
prepopulated_fieldsAuto-fill fields based on others
Pro tip: The @admin.register(Post) decorator is a cleaner alternative to admin.site.register(Post, PostAdmin).
