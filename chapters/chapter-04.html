Working with Templates

In This Chapter

  • Creating the templates directory
  • Rendering templates from views
  • Django Template Language (DTL) basics
  • Template inheritance with extends and block
  • Template filters and tags

Templates separate your presentation logic from your Python code. Let's learn how to build dynamic HTML pages with Django's powerful template engine.

Creating the Templates Directory

Create a templates folder at your project root:

Terminal
mkdir templates

Tell Django where to find your templates by updating settings.py:

settings.py
# settings.py
TEMPLATES = [
    {{
        'BACKEND': 'django.template.backends.django.DjangoTemplates',
        'DIRS': [BASE_DIR / 'templates'],  # Add this line
        'APP_DIRS': True,
        'OPTIONS': {{
            'context_processors': [
                'django.template.context_processors.debug',
                'django.template.context_processors.request',
                'django.contrib.auth.context_processors.auth',
                'django.contrib.messages.context_processors.messages',
            ],
        }},
    }},
]

Rendering Templates from Views

Instead of returning raw HTML strings, use the render() shortcut to render templates:

pages/views.py
# pages/views.py
from django.shortcuts import render

def home(request):
    context = {{
        'title': 'Welcome to Django',
        'items': ['Learn Models', 'Build Views', 'Design Templates'],
    }}
    return render(request, 'home.html', context)

Now create the template file:

templates/home.html
<!-- templates/home.html -->
<!DOCTYPE html>
<html>
<head><title>{{{{ title }}}}</title></head>
<body>
    <h1>{{{{ title }}}}</h1>
    <ul>
        {{% for item in items %}}
            <li>{{{{ item }}}}</li>
        {{% endfor %}}
    </ul>
</body>
</html>

Django Template Language (DTL) Basics

DTL uses special syntax to inject dynamic content into HTML:

SyntaxPurposeExample
{{{{ variable }}}}Output a variable's value{{{{ user.name }}}}
{{% tag %}}Template tags for logic{{% if user.is_authenticated %}}
{{# comment #}}Template comments (not rendered){{# TODO: add nav #}}
{{{{ value|filter }}}}Apply a filter to a value{{{{ name|upper }}}}

Conditionals

template.html
{{% if user.is_authenticated %}}
    <p>Welcome, {{{{ user.username }}}}!</p>
{{% elif user.is_staff %}}
    <p>Welcome, staff member!</p>
{{% else %}}
    <p>Please log in.</p>
{{% endif %}}

Loops

template.html
{{% for task in tasks %}}
    <div class="task">
        <h3>{{{{ task.title }}}}</h3>
        <p>{{{{ task.description }}}}</p>
        {{% if forloop.first %}}<span>First item!</span>{{% endif %}}
    </div>
{{% empty %}}
    <p>No tasks yet.</p>
{{% endfor %}}

Template Inheritance

Template inheritance lets you define a base layout and extend it. This is one of Django's most powerful template features.

templates/base.html
<!-- templates/base.html -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{{% block title %}}My Site{{% endblock %}}</title>
</head>
<body>
    <nav>
        <a href="{{% url 'home' %}}">Home</a>
        <a href="{{% url 'about' %}}">About</a>
    </nav>
    <main>
        {{% block content %}}
        {{% endblock %}}
    </main>
    <footer>&copy; 2026 My Site</footer>
</body>
</html>
templates/home.html
<!-- templates/home.html -->
{{% extends 'base.html' %}}

{{% block title %}}Home{{% endblock %}}

{{% block content %}}
<h1>Welcome Home</h1>
<p>This content replaces the block in base.html.</p>
{{% endblock %}}

Useful Template Filters

FilterWhat It DoesExample
|date:"F d, Y"Formats a date{{{{ post.created|date:"F d, Y" }}}} → March 15, 2026
|truncatewords:30Truncates text to N words{{{{ post.body|truncatewords:30 }}}}
|upperConverts to uppercase{{{{ name|upper }}}} → DJANGO
|default:"N/A"Shows default if value is empty{{{{ bio|default:"No bio yet" }}}}
|lengthReturns the length{{{{ items|length }}}} → 5
