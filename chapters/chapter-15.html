Signals and Custom Managers

In This Chapter

  • What are Django signals?
  • The post_save signal
  • Using the @receiver decorator
  • Connecting signals in apps.py
  • Custom model managers

Signals let different parts of your application communicate when certain actions occur. Custom managers give you reusable query shortcuts.

What Are Signals?

Django signals are a way to execute code automatically when certain events happen — like when a model is saved, deleted, or when a user logs in. They follow the observer pattern: senders notify receivers when something happens.

Common built-in signals:

SignalWhen It Fires
pre_saveBefore a model's save() method
post_saveAfter a model's save() method
pre_deleteBefore a model's delete() method
post_deleteAfter a model's delete() method
m2m_changedWhen a many-to-many relationship changes
user_logged_inWhen a user logs in

The post_save Signal

A common use case: automatically creating a user profile when a new user registers.

accounts/models.py
# accounts/models.py
from django.db import models
from django.contrib.auth.models import User

class Profile(models.Model):
    user = models.OneToOneField(User, on_delete=models.CASCADE)
    bio = models.TextField(blank=True)
    avatar = models.ImageField(upload_to='avatars/', blank=True)
    
    def __str__(self):
        return f'{{self.user.username}} Profile'
accounts/signals.py
# accounts/signals.py
from django.db.models.signals import post_save
from django.dispatch import receiver
from django.contrib.auth.models import User
from .models import Profile

@receiver(post_save, sender=User)
def create_profile(sender, instance, created, **kwargs):
    if created:
        Profile.objects.create(user=instance)

@receiver(post_save, sender=User)
def save_profile(sender, instance, **kwargs):
    instance.profile.save()

Connecting Signals in apps.py

You need to tell Django to load your signals when the app starts:

accounts/apps.py
# accounts/apps.py
from django.apps import AppConfig

class AccountsConfig(AppConfig):
    default_auto_field = 'django.db.models.BigAutoField'
    name = 'accounts'

    def ready(self):
        import accounts.signals  # This connects the signals
Important: Without the ready() method importing your signals module, your signal handlers will never be registered and won't fire.

Custom Model Managers

Custom managers let you add reusable query methods to your models:

tasks/models.py
# tasks/models.py
from django.db import models

class TaskManager(models.Manager):
    def completed(self):
        return self.filter(is_complete=True)

    def pending(self):
        return self.filter(is_complete=False)

    def for_user(self, user):
        return self.filter(owner=user)

class Task(models.Model):
    title = models.CharField(max_length=200)
    is_complete = models.BooleanField(default=False)
    owner = models.ForeignKey('auth.User', on_delete=models.CASCADE)

    objects = TaskManager()  # Replace default manager

    def __str__(self):
        return self.title

Now you can use your custom methods:

Usage
# In views or shell:
Task.objects.completed()           # All completed tasks
Task.objects.pending()             # All pending tasks
Task.objects.for_user(request.user)  # Tasks for current user
