Forms and User Input

In This Chapter

  • Django's form system
  • ModelForm vs regular Form
  • Handling form submissions
  • Form validation
  • Custom widgets

Forms are how users interact with your application. Django provides a powerful forms framework that handles rendering, validation, and security.

Two Types of Forms

Django provides two form base classes:

TypeUse WhenExample
forms.FormYou need a custom form not tied to a modelContact form, search form, login form
forms.ModelFormYou want a form based on a model's fieldsCreate/edit post, user profile, task form

Creating a ModelForm

ModelForms automatically generate form fields from your model:

tasks/forms.py
# tasks/forms.py
from django import forms
from .models import Task

class TaskForm(forms.ModelForm):
    class Meta:
        model = Task
        fields = ['title', 'description', 'due_date', 'is_complete']
        widgets = {{
            'title': forms.TextInput(attrs={{
                'class': 'form-control',
                'placeholder': 'Enter task title'
            }}),
            'description': forms.Textarea(attrs={{
                'class': 'form-control',
                'rows': 4,
                'placeholder': 'Describe your task...'
            }}),
            'due_date': forms.DateInput(attrs={{
                'class': 'form-control',
                'type': 'date'
            }}),
        }}

Creating a Regular Form

contact/forms.py
# contact/forms.py
from django import forms

class ContactForm(forms.Form):
    name = forms.CharField(max_length=100)
    email = forms.EmailField()
    message = forms.CharField(widget=forms.Textarea)
    
    def clean_email(self):
        email = self.cleaned_data['email']
        if not email.endswith('.com'):
            raise forms.ValidationError('Only .com emails accepted.')
        return email

Handling Form Submissions

Here's the standard pattern for processing forms in a view:

tasks/views.py
# tasks/views.py
from django.shortcuts import render, redirect
from .forms import TaskForm

def task_create(request):
    if request.method == 'POST':
        form = TaskForm(request.POST)
        if form.is_valid():
            task = form.save(commit=False)
            task.owner = request.user
            task.save()
            return redirect('task_list')
    else:
        form = TaskForm()
    
    return render(request, 'tasks/task_form.html', {{'form': form}})

Rendering Forms in Templates

templates/tasks/task_form.html
<!-- templates/tasks/task_form.html -->
{{% extends 'base.html' %}}

{{% block content %}}
<h1>Create Task</h1>
<form method="POST">
    {{% csrf_token %}}
    {{{{ form.as_p }}}}
    <button type="submit">Save Task</button>
</form>
{{% endblock %}}
Security: Always include {{% csrf_token %}} in your forms. This protects against Cross-Site Request Forgery attacks. Django will reject POST requests without a valid CSRF token.

Form Rendering Options

MethodRenders As
form.as_pEach field wrapped in <p> tags
form.as_tableEach field as a table row
form.as_ulEach field as a list item
Manual renderingFull control over each field's HTML

For manual rendering with full control:

template.html
<form method="POST">
    {{% csrf_token %}}
    <div class="field">
        <label for="{{{{ form.title.id_for_label }}}}">Title</label>
        {{{{ form.title }}}}
        {{% if form.title.errors %}}
            <span class="error">{{{{ form.title.errors.0 }}}}</span>
        {{% endif %}}
    </div>
    <button type="submit">Save</button>
</form>
