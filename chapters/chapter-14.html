API Authentication & Permissions

In This Chapter

  • Why API authentication matters
  • Token authentication setup
  • Obtaining and using tokens
  • Permission classes
  • Custom permissions

APIs need authentication and permissions to control who can access what. Let's secure our API with token authentication.

Why API Authentication Matters

Without authentication, anyone can read, create, modify, or delete your data through the API. DRF provides several authentication schemes:

  • Session Authentication — Uses Django's session framework (browser-based)
  • Token Authentication — Uses a unique token per user (API clients)
  • JWT Authentication — JSON Web Tokens (stateless, popular for SPAs)

Setting Up Token Authentication

helloworld/settings.py
# helloworld/settings.py
INSTALLED_APPS = [
    # ... other apps ...
    'rest_framework',
    'rest_framework.authtoken',  # Add this
]

REST_FRAMEWORK = {{
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'rest_framework.authentication.TokenAuthentication',
        'rest_framework.authentication.SessionAuthentication',
    ],
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.IsAuthenticated',
    ],
}}

Run migrations to create the token table:

Terminal
python manage.py migrate

Obtaining a Token

Add the token endpoint to your URLs:

helloworld/urls.py
# helloworld/urls.py
from rest_framework.authtoken.views import obtain_auth_token

urlpatterns = [
    # ... other patterns ...
    path('api/token/', obtain_auth_token, name='api_token'),
]

Users can now obtain a token by sending their credentials:

Terminal
# Request a token
curl -X POST http://127.0.0.1:8000/api/token/ \
    -d "username=admin&password=secretpass"

# Response:
# {{"token": "9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b"}}

Using the Token in Requests

Include the token in the Authorization header:

Terminal
# Authenticated API request
curl -H "Authorization: Token 9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b" \
    http://127.0.0.1:8000/api/books/

Permission Classes

DRF provides built-in permission classes:

PermissionWho Can Access
AllowAnyAnyone, including unauthenticated users
IsAuthenticatedOnly authenticated users
IsAdminUserOnly admin/staff users
IsAuthenticatedOrReadOnlyAuthenticated for write, anyone for read

Apply permissions per-view:

views.py
from rest_framework.permissions import IsAuthenticated, IsAdminUser

class BookViewSet(viewsets.ModelViewSet):
    queryset = Book.objects.all()
    serializer_class = BookSerializer
    
    def get_permissions(self):
        if self.action in ['list', 'retrieve']:
            return [IsAuthenticated()]
        return [IsAdminUser()]

Custom Permissions

permissions.py
# permissions.py
from rest_framework.permissions import BasePermission

class IsOwner(BasePermission):
    def has_object_permission(self, request, view, obj):
        return obj.owner == request.user
